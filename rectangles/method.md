# 메서드 문법 (method syntax)
> 메서드는 함수와 유사하지만 구조체 콘텍스트에 정의된다.( 열거형, 트레이트 객체 안에서도 정의됨)<br>
> 메서드의 첫 번째 매개변수는 항상 self 가 들어간다. self 매개변수는 메서드를 호출하고 있는 구조체 인스턴스를 나타낸다.<br>
> 메서드 문법은 인스턴스, 점(.), 메서드명, 괄호 및 인수 로 구성된다
```
rect1.area()
```
> &self 는 self: &Self 를 줄인 것 이다. impl 블록 내, Self 는 impl 블록의 대상이 되는 타입의 별칭(별명)이다.<br>
> 메서드는 Self 타입의 self 라는 이름의 매개변수를 첫 번째 매개변수로 가져야 한다. 그래야 self 형태의 축약형을 사용할 수 있기 때문이다.<br>
> 메서드가 Self 의 인스턴스를 빌려온다는 것을 나타내기 위해 self 축약형 앞에 & 를 붙인다. 다른 매개변수가 그런 것 처럼 self 의 소유권을 가져올수도, 불변으로 self 를 빌려올 수도, 가변으로 가져올 수 있다.<br>
> 데이터를 읽기만 할땐, 불변으로 self 를 빌려온다. 만약 메서드에서 작업 중 호출한 인스턴스를 변경하고 싶다면, 첫 번째 매개변수로 &mut self 를 사용하면 된다.<br><br>
> 구조체의 필드 이름과 동일한 이름의 메서드를 만들 수도 있다. 이는 괄호() 가 붙어있으면 메서드로 인식하고 괄호가 없으면 필드로 인식한다.<br>
> 그러나 필드와 동일한 이름의 메서드를 만드는 경우는 해당 필드의 값을 얻어올 때만 사용된다. 이를 게터(getter) 라고 부른다.<br>
> Rust 는 구조체 필드에 대한 getter 를 자동으로 만들지 않는다. 필드를 비공개(private)로 하고 메서드는 공개(public)로 만들 수 있기 때문에, getter 는 어떤 타입의 공개 API 로써 어떤 필드에 대해 읽기 전용 접근만 허용하고자 할때 유용하다.<br>


# 연관 함수(associated function)
> impl {} 블록 내 구현된 모든 함수를 연관 함수라 부른다. 이는 impl 뒤에 나오는 타입과 모두 연관된 함수이기 때문이다.<br>
> 작동하는데 해당 타입의 인스턴스가 필요하지 않다면, self 를 첫 매개변수로 갖지 않는(메서드가 아닌)연관 함수를 정의할 수 있다.<br><br>
> 메서드가 아닌 연관 함수는 구조체의 새 인스턴스를 반환하는 생성자로 자주 활용된다. 이 함수들은 보통 new 라고 불리는데, new 는 Rust 에서 특별한 이름 또는 키워드가 아니다.<br>
> 연관 함수는 특정 구조체에 속하지만, 인스턴스를 필요로 하지 않는 함수이다. 즉, 매개변수로 &self 를 사용하지 않음. 

## 여러개의 impl 블록
> 구조체는 여러개의 impl 블록을 가질 수 있다. 즉, impl 블록을 반드시 하나만 작성해야 할 필요는 없다.<br>

## 정리
> 구조체를 사용하면 도메인에 의미 있는 커스텀 타입을 만들 수 있다.<br>
> 구조체를 사용함으로써, 서로 관련 있는 데이터들을 하나로 묶어 관리할 수 있다. 또, 각 데이터 조각에 이름을 붙여 코드를 더 명확하게 만들 수 있다.<br>
> impl 블록 내 타입에 대한 연관 함수들, 연관 함수의 일종인 메서드를 정의하여 구조체 인스턴스가 가질 동작들을 명시할 수 있다.<br>
